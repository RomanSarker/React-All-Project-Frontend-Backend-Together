Tell us the use cases of null and undefined:

null is used to explicitly represent the absence of a value or to clean up references. undefined is often unintentional and represents the absence of a defined value or variable. null is typically used when you want to convey the deliberate absence of a value, while undefined usually indicates an unintentional absence or uninitialized variable. Both are falsy values, but they serve slightly different purposes in JavaScript.

What do you mean by REST API?:

REST API (Representational State Transfer Application Programming Interface) is a set of rules and principles for creating web services that use standard HTTP methods and follow a client-server architecture. It emphasizes statelessness, resources, and a uniform interface for easy and scalable communication between clients and servers.
